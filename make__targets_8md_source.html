1 Makefile Targets {#makefile_targets}
2 ====================================
3 
4 While mavtables uses [CMake](https://cmake.org/) for building a simple Makefile
5 (which calls `cmake`) is included for convenience. The available targets are
6 listed below.
7 
8 
9 ## release (default)
10 
11 Build `mavtables` using the _Release_ option which generates an optimized (for
12 speed) executable without debug symbols. `mavtables` is placed at
13 `build/mavtables`.
14 
15 Unit tests are built as well under `build/unit_tests` and are built with
16 optimizations.
17 
18 
19 ## debug
20 
21 Build `mavtables` and unit tests using the _Debug_ option which will enable
22 debug flags and turn off optimizations. The executables are located at
23 `build/mavtables` and `build/unit_tests`.
24 
25 
26 ## test
27 
28 Build and run all tests. Currently this is just unit tests at
29 `build/unit_tests` which are built with the `debug` target above.
30 
31 
32 ## coverage
33 
34 Build and run all tests and compute test coverage. This option requires
35 [lcov](http://ltp.sourceforge.net/coverage/lcov.php) and
36 [gcovr](http://gcovr.com/) to be installed and will only work when the compiler
37 is `g++`.
38 
39 The coverage per file will be printed to the terminal and a detailed, line by
40 line, report generated with [lcov](http://ltp.sourceforge.net/coverage/lcov.php)
41 at `build/lcov/html/selected_targets/index.html`.
42 
43 
44 ## style
45 
46 Fix the style of C++ source code and header files. The original files are
47 backed up with a `.orig` extension. [Artistic
48 Style](http://astyle.sourceforge.net/) is required to use this target.
49 
50 
51 ## doc
52 
53 Generate html documentation with
54 [Doxygen](http://www.stack.nl/~dimitri/doxygen/) and place it at
55 `build/doc/html/index.html`.
56 
57 
58 ## gh-pages
59 
60 Generate html documentation and publish to
61 [https://shamuproject.github.io/mavtables](https://shamuproject.github.io/mavtables).
